# 馬達—螺旋槳負載匹配理論
### 從電機特性線到實際懸停平衡點 — 以 CL-8520 / CL-8020 為例

適用對象:55g 級四軸機架(對角軸距 120mm)、1S LiPo(3.7V 標稱)、有刷空芯杯馬達 + 直驅小槳。

---

## 0. 問題定義

型錄上的「最大效率點」是測功機在固定電壓下對馬達掃描任意機械負載找到的效率峰值,**只描述馬達本身**,跟裝上某顆螺旋槳之後系統真正會停在哪裡無關。本文推導真正決定運轉點的方程,並用它系統性比較 CL-8520(8 種繞線)與 CL-8020(4 種繞線)在 3.7V 下搭配 45/50/55/60/65mm 槳的表現,回答兩個具體問題:

1. CL-8020 KV14.9 是不是最佳選擇?
2. 槳徑該怎麼選——50mm、60mm 會不會更好?

---

## 1. 馬達電機模型

含摩擦損耗的有刷直流馬達:

```
電壓方程:   V = I·R + Kt·ω
軸端輸出扭矩: T(ω) = Kt·(I − I₀)
```

解出電流並代入扭矩式,得到馬達在固定電壓下的 T-ω 特性——**一條直線**:

```
T_motor(ω) = Kt·(V/R − I₀) − (Kt²/R)·ω  ≡  a − b·ω

a = Kt(V/R − I₀)   ……該電壓下的軸端堵轉扭矩
b = Kt²/R          ……直線斜率(與電壓無關)
```

### 1.1 參數反推與型錄自洽驗證

每一列型錄資料本身就包含三個操作點(空載、最大效率、堵轉),可直接反推 Kt、R、I₀:

```
Kt ≈ V_nom / ω₀(空載,rad/s)
R  = V_nom / I_堵轉
I₀ = 空載電流(直接讀表)
```

以 CL-8520 KV13.7(V_nom=3.7V)為例驗證:
- Kt = 3.7 / (50700rpm→5309rad/s) = 6.97×10⁻⁴ V·s/rad
- R = 3.7 / 4.47A = 0.828Ω(型錄端電阻標 0.83Ω,吻合)
- 反算堵轉扭矩 a = Kt(V/R−I₀) = 3.04 mN·m(型錄標 2.975 mN·m,差 2%,在量測誤差內)

這套方法可以套用在任何一列,用「型錄端電阻欄」作為自洽性檢查。

### 1.2 跨電壓延伸

Kt、R、I₀ 是繞線的物理常數,與外加電壓無關。只要落在「工作電壓範圍」內,同一組 (Kt,R,I₀) 可以用在型錄測試電壓以外的任何電壓——這正是本文評估全部 12 種繞線在統一的 3.7V(1S)下表現的依據。但離型錄測試電壓越遠,外推的可信度越低(見第 9 節)。

---

## 2. 螺旋槳氣動負載模型

標準無因次係數定義(n = 轉速 rev/s,D = 直徑 m,ρ = 空氣密度):

```
推力:F = C_T·ρ·n²·D⁴
扭矩:Q = C_Q·ρ·n²·D⁵   (C_Q = C_P/2π)
```

以角速度 ω=2πn 表示:

```
Q_prop(ω) = k_Q·ω²,   k_Q = C_P·ρ·D⁵/(4π²)
F_prop(ω) = k_T·ω²,   k_T = C_T·ρ·D⁴/(4π²)
```

**這是一條過原點的二次拋物線**——跟馬達的線性直線是完全不同的數學形式。低雷諾數微型槳(45–65mm,對應 UIUC/Deters 等文獻研究的 2.25–5in 範圍)的代表性係數約 C_T≈0.08–0.12、C_P≈0.045–0.07,本文計算統一取 **C_T=0.10、C_P=0.055** 作為示範值(非任何特定品牌槳的實測值,見第 10 節限制)。

---

## 3. 平衡點方程

令馬達輸出扭矩 = 槳的需求扭矩:

```
a − b·ω = k_Q·ω²
→ k_Q·ω² + b·ω − a = 0
→ ω* = [−b + √(b² + 4·k_Q·a)] / (2·k_Q)
```

解出 ω* 後:

```
實際推力:F* = k_T·ω*²
實際電流:I* = (V − Kt·ω*)/R
實際效率:η* = (k_Q·ω*²·ω*) / (V·I*)
```

**這個 (ω*, T*, F*, I*, η*) 才是設計基準——不是型錄的最大效率點。**

---

## 4. 為什麼最大效率點不能當基準(關鍵定理)

馬達自身曲線上存在一個**絕對效率峰值**,與任何槳無關。設 u=ω/ω₀(ω₀=V/Kt,本電壓下的空載轉速),γ=I₀·R/V=I₀/I_堵轉(摩擦電流占堵轉電流的比例),代入 η(u)=T(u)·u·ω₀/(V·I(u)) 並化簡,可得到優美的封閉解:

```
η(u) = u − γ·u/(1−u)

令 dη/du = 0  →  u* = 1 − √γ
此時效率上限: η_max = u*² = (1 − √γ)²
```

**驗證**(以本文三個代表性繞線在 3.7V 下計算):

| 繞線 | γ=I₀R/V | u*=1−√γ | η_max=(1−√γ)² | 型錄標示 η_max |
|---|---|---|---|---|
| CL-8520 KV14.0 | 0.01702 | 86.9% | 75.6% | 73.12%(@3.0V 測試) |
| CL-8520 KV10.9 | 0.01575 | 87.5% | 76.5% | 74.83%(@3.2V 測試) |
| CL-8020 KV14.9 | 0.01933 | 86.1% | **74.13%** | **74.13%**(@3.7V 測試) |

CL-8020 KV14.9 因為測試電壓恰好就是 3.7V(零外推),算出來的理論上限與型錄數字**完全吻合**——這直接證實:**型錄的「最大效率點」就是這條公式算出的、馬達自身曲線的絕對峰值,跟裝什麼槳無關**。它只在「剛好挑到一顆讓平衡點落在 u*=1−√γ 附近的槳」時才會被實際觸及,這是巧合,不是規則。

---

## 5. 正確設計流程

1. 訂目標推力 F_total=m·g,每馬達 F_motor=F_total/4,依需求乘上懸停餘量係數 k(自由飛抗擾常取 1.5–2.5;純繫留 relay 測試可低一些,但要留振盪空間)。
2. 取得槳的 C_T、C_Q(實測最準,理論估算次之)。
3. 對每個候選馬達×槳×電壓,解第 3 節方程得到 (ω*,T*,F*,I*,η*)。
4. 檢查 F*×4 ≥ F_total×k(推力是否夠)。
5. 檢查 I*、I²R 損耗(熱安全)。
6. 用第 4 節的 η_max=(1−√γ)² 當作該繞線的效率天花板,評估 η* 與天花板的差距,作為「這顆槳跟這顆馬達搭不搭」的量化指標。

## 7. 帕雷托分析——CL-8020 KV14.9 是否最佳?

### 7.1 第一輪結論(全部 12 種繞線,允許跨電壓外推)

對 60 組合做帕雷托最優分析,CL-8020 KV14.9 的全部 5 個槳徑點都被某個 CL-8520 組合超越(例如 KV14.0+50mm 在完全相同的 T/W=2.55 下,效率 72.3% 比 KV14.9+50mm 的 68.4% 高近 4 個百分點)。帕雷托邊界完全由 CL-8520(KV10.9/14.0/16.0)構成。

**但這個結論有一個必須澄清的前提,以下修正。**

### 7.2 關鍵澄清:「額定電壓」≠「工作電壓上限」

型錄裡的「額定電壓 V_nom」只是廠商選擇**發表那三個操作點數據**(空載/最大效率/堵轉)時用的**單一測試電壓**,不是這顆繞線能用的電壓上限——真正的硬限制是「工作電壓範圍」欄(CL-8520 的 1.5–4.0V 或 1.5–3.7V,CL-8020 全系列的 1.5–4.0V)。3.7V 對所有 12 種繞線而言**都在合法工作範圍內**,沒有任何一顆是「超規格」使用。

真正的風險不是「能不能用」,而是「模型準不準」:第 1 節的 Kt、R 是用 V_nom 那一個測試點反推出來的,拿去預測**另一個電壓**(3.7V)下的行為時,假設了 Kt、R、I₀ 三者不隨電壓變化。Kt、R 是繞線匝數/線徑/磁鋼決定的物理常數,這個假設很可靠;I₀(摩擦/風阻電流)理論上可能隨轉速有二階變化,這部分才是真正的不確定性來源,而且**外推幅度越大,這個二階效應累積的誤差就可能越大**。

CL-8520 KV14.0 額定 3.0V、KV16.0 額定 2.4V,用在 3.7V 上分別外推了 23%、54%——這些預測在物理上是合理的,但缺乏直接證據。**CL-8020 KV14.9 額定電壓恰好就是 3.7V,等於外推幅度為零,是唯一不需要做任何電壓外推假設就能直接信任的型錄數字。**

### 7.3 第二輪結論:只比較「零外推」的繞線

12 種繞線裡,額定電壓剛好等於 3.7V 的只有三顆:CL-8520 KV11.9、CL-8520 KV13.7、CL-8020 KV14.9。把比較範圍**收斂到這三顆**(每一個數字都直接來自型錄,不依賴任何跨電壓假設)重新做帕雷托分析:

| 繞線 | 45mm | 50mm | 55mm | 60mm | 65mm |
|---|---|---|---|---|---|
| 8520 KV11.9 | T/W1.26 η72.0% | T/W1.58 η67.4% | T/W1.87 η61.5% | T/W2.09 η55.3% | T/W2.25 η49.2% |
| 8520 KV13.7 | T/W1.41 η67.3% | T/W1.70 η61.1% | T/W1.93 η54.5% | T/W2.08 η48.0% | T/W2.18 η42.1% |
| **8020 KV14.9** | **T/W2.01 η72.7%** | **T/W2.55 η68.4%** | **T/W3.03 η62.7%** | **T/W3.41 η56.5%** | **T/W3.69 η50.5%** |

結果非常乾淨:**KV11.9 與 KV13.7 的全部 10 個點,每一個都同時被 KV14.9 的某個點在 T/W 與效率兩個維度上超越**(例如 KV11.9+60mm 的 T/W=2.09、η=55.3%,被 KV14.9+50mm 的 T/W=2.55、η=68.4% 雙重碾壓)。**KV14.9 的全部 5 個點都在這個限定範圍內構成完整的效率邊界。**

### 7.4 修正後的最終結論

**如果把比較限定在「不需要跨電壓外推、可以直接信任型錄數字」的範圍內,CL-8020 KV14.9 就是最佳選擇**——而且槳徑同樣是 45mm 最優(η=72.7%,非常接近第 4 節算出的該繞線理論天花板 74.13%,已達天花板的 98%)。

第一輪結論裡看起來更好的 CL-8520 KV14.0、KV10.9、KV16.0 並沒有被推翻——它們在物理上仍然合理,如果實測證實 Kt/R/I₀ 確實不隨電壓改變,KV14.0+50mm 應該真的會比 KV14.9+50mm 效率高。但**在沒有實測驗證之前,這只是一個有模型風險的預測,不是可以直接信任的結論**。在電池電壓已經固定、且手上已經有一顆零外推風險的選項時,合理的工程判斷是:**先用 CL-8020 KV14.9 + 45mm,除非實際把 CL-8520 KV14.0 拿上測試臺量出來確實更好。**

## 8. 槳徑該怎麼選——50mm、60mm 會更好嗎?

**不會。** 在「全部槳徑使用同一組 C_T/C_P」的假設下,12 種繞線**無一例外**呈現同一趨勢:槳徑從 45→50→55→60→65mm,效率單調下降、電流與發熱單調上升,T/W 單調上升。也就是說在這個模型裡,槳徑與效率/熱安全之間**沒有局部極值**,只有單調的「用效率換推力」的交換關係——50mm、60mm 不會「免費」變好,只會用更多電流/更多發熱換來更多推力餘量。

但這不代表「越小越好」沒有極限。第 4 節證明每條馬達線上有一個絕對效率峰值,出現在 u*=ω/ω₀=1−√γ(本案例約 86–87% 空載轉速)。如果槳太小,平衡點會被推得比這個峰值還靠近空載端(轉速更高,摩擦損耗占比反而回升),效率同樣會下降——只是在 45–65mm 這個測試範圍裡,所有組合的平衡點都還落在峰值的「重負載側」(u<u*),所以才會看到單調趨勢。換句話說:**45mm 之所以在本文範圍內最好,只是因為它是測試範圍裡最接近 u* 的點,不是因為「小」本身有特殊性。**

## 9. 模型限制(務必注意)

1. **C_T、C_P 假設與真實槳不符**:本文對 45–65mm 五種直徑統一取相同係數,但真實小槳的氣動效率會隨直徑變化(雷諾數效應),通常**直徑越小、Re 越低、氣動效率(figure of merit)越差**。這代表電氣側分析雖然偏好小槳,氣動側可能反過來偏好稍大的槳——真正的甜蜜點需要每個直徑各自的 C_T(D)、C_P(D) 實測值才能確定,本文的「45mm 最好」結論可能因此被高估。
2. **電壓外推風險**:KV=16.0/17.7/18.0(額定 2.4–3.2V)外推到 3.7V 幅度達 31–54%,Kt/R/I₀ 假設為電壓無關常數,實際馬達在這麼大的外推範圍可能因渦流損耗、軸承摩擦隨轉速變化等二階效應而偏離線性模型。KV=14.0(23%外推)、KV=10.9(16%外推)風險較低,KV=11.9/13.7(零外推,額定剛好3.7V)風險最低但本文計算中效率較不突出。
3. **摩擦電流 I₀ 假設為常數**:實際上 I₀ 可能隨轉速略增(風阻、軸承黏滯),本文簡化為固定值。
4. **電壓下垂未計入**:1S 電池由 4.2V 放到 3.5V 過程中,a(V) 隨 V 線性下降、b 不變,平衡點會沿新的馬達線往左下移動。長時間繫留測試若要精確比較,應把實際量到的電壓代入重新求解。
5. **未含安裝損耗**:槳座偏心、軸承摩擦、ESC/驅動電路壓降等均未計入。

## 10. 建議的實測驗證流程

繫留測試臺本身就是最準的量測工具:固定電壓做一次 PWM 階梯式 sweep,同步記錄 RPM、電流、(若有秤重)推力,即可直接畫出每顆候選槳真實的 F(ω)、Q(ω) 曲線,完全不需要假設 C_T、C_P。建議步驟:

1. 對每顆候選槳(45/50/55/60/65mm,以及若有意願可加測 40mm)在固定電壓下做 5–8 個 PWM 階梯。
2. 每個階梯穩定後記錄:RPM、電流、(若有)推力。
3. 用最小平方法對 (ω,Q) 數據擬合 Q=k_Q·ω²,驗證是否真為二次關係(若否,代表 C_Q 隨 RPM 變化,需用分段或更高階模型)。
4. 把擬合出的 k_Q、k_T 代回第 3 節方程,與第 6 節的理論表對照,看哪個候選馬達+槳的實測效率最接近第 4 節算出的 η_max=(1−√γ)² 天花板。

## 附錄 B:Python 計算程式(可重現)

import math

rho, CT, CP = 1.2, 0.10, 0.055
CQ = CP / (2*math.pi)
V_drive = 3.7
m, g = 0.055, 9.81
Weight = m*g

# (series, KV標籤, V_nom, V_min, V_max, I0_mA, w0_rpm@Vnom, I_堵轉@Vnom, 單顆重量g)
motors = [
    ("8520","10.9",3.2,1.5,4.0,120.0,35000,6.59,4.99),
    ("8520","11.9",3.7,1.5,4.0,92.0,44000,4.61,4.99),
    ("8520","12.6",3.4,1.5,4.0,140.0,43000,6.90,4.99),
    ("8520","13.7",3.7,1.5,4.0,102.0,50700,4.47,4.99),
    ("8520","14.0",3.0,1.5,4.0,190.0,42000,9.05,4.99),
    ("8520","16.0",2.4,1.5,3.7,170.0,38500,8.00,4.99),
    ("8520","17.7",2.4,1.5,3.7,280.0,42500,7.47,4.99),
    ("8520","18.0",3.2,1.5,3.7,270.0,57500,10.67,4.99),
    ("8020","7.1", 3.0,1.5,4.0,75.0,21300,2.00,4.69),
    ("8020","11.7",3.0,1.5,4.0,120.0,35000,5.10,4.69),
    ("8020","13.6",3.0,1.5,4.0,140.0,40800,6.98,4.69),
    ("8020","14.9",3.7,1.5,4.0,187.5,55000,9.70,4.69),
]
props_mm = [45,50,55,60,65]

def kTkQ(D):
    kT = CT*rho*(D**4)/(4*math.pi**2)
    kQ = CQ*rho*(D**5)/(4*math.pi**2)
    return kT, kQ

results = []
for series, kv, Vnom, Vmin, Vmax, I0_mA, w0rpm, Is, wt in motors:
    if not (Vmin-1e-9 <= V_drive <= Vmax+1e-9):
        continue
    I0 = I0_mA/1000.0
    w0_test = w0rpm*math.pi/30
    Kt = Vnom/w0_test
    R  = Vnom/Is
    a  = Kt*(V_drive/R - I0)
    b  = (Kt**2)/R
    w0_drive = V_drive/Kt
    Istall_drive = V_drive/R
    for Dmm in props_mm:
        D = Dmm/1000.0
        kT, kQ = kTkQ(D)
        disc = b*b + 4*kQ*a
        w = (-b + math.sqrt(disc)) / (2*kQ)
        if w < 0 or w > w0_drive:

        F = kT*w*w

        TW = (4*F)/Weight

# 帕雷托最優過濾(同一T/W下找不到效率更高者)
def is_pareto(r, all_r):
    return not any(s is not r and s['TW']>=r['TW'] and s['eff']>=r['eff']
                   and (s['TW']>r['TW'] or s['eff']>r['eff']) for s in all_r)

pareto = [r for r in results if is_pareto(r, results)]

for r in sorted(results, key=lambda r:(r['series'],r['kv'],r['D'])):
    print(f"{r['series']}-{r['kv']:5s} {r['D']:3d}mm  rpm={r['rpm']:7.0f}  "

          f"Ploss={r['Ploss']:.2f}W  T/W={r['TW']:.2f}")

## 附錄 C:如何看懂 Coreless Motor 特性曲線型錄

(對應廠商提供的 CL-8020-1/-2、CL-8520-1/-2 四張連續特性曲線圖,見第 9.1 節的逐圖核對)

型錄表格只給三個點(空載、最大效率、堵轉),但廠商通常還會附一張**連續曲線圖**(Speed/Eff/Watts/Amp 對 Torque),把整條曲線都畫出來。這張圖資訊量遠大於三個點,但格式對第一次看的人很不友善——尤其是「共用右軸」的設計。本附錄把圖上每一條線、每一個轉折,都對回第 1、3、4 節已經推導過的方程式。

### C.1 這張圖在量什麼

廠商把馬達架在測功機(dynamometer)上,**固定一個電壓**,用可調的機械刹車(磁滯刹車或渦電流刹車)從「完全不施加負載」一路加到「轉速被壓到 0(堵轉)」,逐點記錄轉速與電流,再算出輸出功率與效率。

**關鍵認知**:橫軸的扭矩 T 是測功機外加的**任意**機械負載,不是任何特定螺旋槳的負載。第 2 節說過,槳的負載是 Q=k_Q·ω² 這條通過原點的拋物線——型錄圖上不會幫你畫這條線,你必須自己疊上去,跟馬達線的交點才是真正的運轉平衡點(第 3 節)。這張型錄圖能告訴你的,是「這顆馬達在任何可能的負載下會怎麼反應」,不是「這顆馬達裝上某顆槳會停在哪裡」。

### C.2 看懂雙 Y 軸 ——最容易搞混的地方

圖上有兩條 Y 軸:
- **左軸**:轉速(RPM),有自己真正的刻度,藍色虛線(Speed)直接照這個軸讀。
- **右軸**:0–100% 的「共用」軸,**同時**給效率(Eff%)、功率(Watts)、電流(Amp)三條曲線使用,但三者的「滿格 100% 代表的實際值」各自不同,印在圖例的「Full scale」欄裡(例如 CL-8020-1 標示 `Eff 100% / Watts 5.0 / Amp 5.0`)。

讀數規則:
```
Eff 曲線:右軸刻度本身就是百分比,直接讀。
Watts 曲線:實際瓦特 = (右軸讀到的高度 ÷ 100%) × Full Scale Watts
Amp 曲線:  實際安培 = (右軸讀到的高度 ÷ 100%) × Full Scale Amp
```
例如某張圖標示 `Full scale: 5.0 Watts`,若綠色功率線在某個扭矩下的高度是右軸 70%,實際功率就是 0.70×5.0 = 3.5W——**不是「70」也不是「70%的某個你以為的值」,一定要回頭看 Full Scale 數字**。這是初學者最常出錯的地方,因為三條完全不同物理量的曲線疊在同一條視覺軸上。

### C.3 四條曲線,對照第 1、3、4 節的方程式

把第 1 節的馬達方程 `V=IR+Kt·ω`、`T=Kt(I−I₀)` 整理成「以 T 為自變數」的形式(跟型錄橫軸一致),四條曲線就是四個函數:

**① 轉速線(藍色,直線)**
```
ω(T) = (a − T)/b
```
直接從第 1 節 `T=a−bω` 解出來。y 截距(T=0)是 ω₀=V/Kt;x 截距(ω=0)是 T_stall=a。**這條線是直的,因為馬達電壓方程本來就是線性的**——這也是本文整套理論的起點。

**② 電流線(黑色,直線,且比轉速線更簡單)**
```
I(T) = T/Kt + I₀
```
這直接就是 `T=Kt(I−I₀)` 的代數重排,**完全不需要 V 或 R**,只跟扭矩常數 Kt 與摩擦電流 I₀ 有關。y 截距是 I₀;在 T=T_stall 處的值是 I_stall=V/R。電流隨扭矩線性上升,是這四條線裡最直觀的一條。

**③ 功率線(綠色,開口向下的拋物線)**
```
P(T) = T·ω(T) = (aT − T²)/b
```
這是輸出**機械**功率(T 乘轉速),不是輸入電功率(電功率 V·I(T) 其實是直線,因為 I(T) 是直線——但型錄畫的 Watts 通常是指這條機械輸出功率拋物線)。對 T 微分設為 0:
```
dP/dT = (a−2T)/b = 0  →  T = a/2
P_peak = a²/(4b)
```
**功率峰值永遠出現在堵轉扭矩的一半**,跟 Kt、R、I₀ 的具體數值無關,這是線性馬達模型的通用結論。

**④ 效率線(洋紅色,鐘形,峰值比功率峰值更靠近空載端)**
```
η(T) = P(T) / (V·I(T))
```
代入 u=ω/ω₀ 整理後,就是第 4 節推出的封閉解:
```
η(u) = u − γu/(1−u),   γ = I₀R/V
峰值在 u* = 1−√γ,對應 η_max = (1−√γ)²
```
u* 換算回扭矩:T* ≈ a(1−u*) 附近(精確值見第 4 節推導),由於 γ 通常只有 0.01–0.03,√γ 很小,u* 會非常接近 1(也就是接近空載端),**遠比功率峰值的 u=0.5 更靠左**。這正是型錄圖上「洋紅色效率曲線的峰值,看起來比綠色功率曲線的峰值更早出現(扭矩更小)」的數學原因——兩條鐘形曲線的峰值天生就不在同一個位置。

### C.4 不靠表格的三個點,直接從一張圖反推 Kt、R、I₀

如果手上只有曲線圖、沒有型錄表格,可以這樣重建整套模型:

1. 讀藍色速度線在 T=0 的高度(左軸直接讀)→ ω₀,算出 Kt≈V/ω₀(rad/s)。
2. 讀黑色電流線在 T=0 的高度(右軸讀,記得乘 Full Scale Amp)→ I₀。
3. 讀速度線跌到 0 的橫軸位置 → T_stall=a;讀電流線在同一橫軸位置的高度(乘 Full Scale)→ I_stall,算出 R=V/I_stall。
4. **交叉驗證 1**:綠色功率線的峰值橫軸位置應該正好是 T_stall/2——如果明顯不是,代表這張圖可能還有本文模型沒蓋到的損耗機制。
5. **交叉驗證 2**:讀洋紅色效率峰值的高度 η_max,反算 γ=(1−√η_max)²,檢查是否等於用第 1–3 步算出的 I₀R/V——兩者吻合,代表整套線性模型在這張圖的範圍內是自洽的。

第 9.1 節對 CL-8020-1/-2、CL-8520-1/-2 四張圖做的逐圖核對,就是完整跑過這五步,而且四張圖、每張圖三個獨立特徵全部命中型錄表格數字。

### C.5 常見誤讀陷阱

- **忘記乘 Full Scale**:把右軸的「看起來幾%」直接當成瓦特或安培數值。三條曲線共用一條軸,但滿格代表的實際值不同,一定要回頭看圖例。
- **把堵轉點當作可連續使用的工作點**:堵轉電流(I_stall)是測試瞬間量到的值,不是連續額定電流;真實使用時電流長時間停在 I_stall 附近會嚴重過熱(對應第 9 節的熱安全討論)。
- **效率峰值跟功率峰值混為一談**:兩者位置不同(u*=1−√γ 比 u=0.5 更靠空載端),不要拿其中一個的橫軸位置去估另一個。
- **不同電壓的圖直接比較斜率或截距**:斜率 b=Kt²/R 雖然跟電壓無關,但截距 a=Kt(V/R−I₀) 隨 V 變化;比較兩張不同電壓的圖之前,必須先用本文方法統一換算到同一個電壓下的模型,而不是直接比較圖上的數字。
- **把「額定電壓」誤認為唯一合法電壓**:額定電壓只是型錄選擇發表測試點的那一個電壓,不是工作電壓的唯一值——真正的限制是「工作電壓範圍」欄(對照第 7.2 節的澄清)。
