## 0. 核心問題定義

有三個問題新手設計師常常搞混:

1. **為什麼不能拿型錄的「最大效率點」直接當懸停工作點?**
2. **馬達+槳的系統最終「落在哪裡」是由什麼決定的?**
3. **型錄中的額定電壓、KV 值、端電阻,如何換算成真實飛行中的推力與電流?**

這三個問題的答案都來自同一個框架:**馬達的線性 T-ω 特性線**與**螺旋槳的二次拋物線負載曲線**的交點理論(平衡工作點,Operating Point)。以下完整推導。

## 1. 馬達電機模型

### 1.1 為什麼轉速-轉矩是線性關係?

空芯杯馬達本質上是直流刷馬達(DC Brushed Motor),其線性關係來自兩個核心物理公式:

**①電流產生轉矩**(電磁感應定律):
```
τ = Kt · I  →  I = τ/Kt
```

**②旋轉產生反電動勢(Back EMF)**:
```
E = Ke · ω  (國際單位制中 Ke = Kt)
```

**③電壓迴路平衡**:
```
V − E = I · R  →  V − Kt·ω = I·R
```

三式聯立,消去電流 I:
```
V − Kt·ω = (τ/Kt)·R
→  ω = V/Kt − (R/Kt²)·τ
```

因為固定電壓 V 下,V/Kt 是常數(空載轉速 ω₀),R/Kt² 也是常數(斜率),這就是標準直線方程式 **y = b − mx**——這就是馬達 T-ω 曲線必然是直線的物理根源。

### 1.2 含摩擦損耗的完整模型

真實馬達存在軸承摩擦與電刷阻力,用**摩擦電流 I₀**(空載電流)建模:

```
軸端輸出扭矩:T(ω) = Kt·(I − I₀)
```

代入電壓方程,得到含摩擦的馬達特性直線:

```
T_motor(ω) = [Kt·V/R − Kt·I₀] − (Kt²/R)·ω  ≡  a − b·ω
```

其中:
```
a = Kt(V/R − I₀)    ── 該電壓下的軸端堵轉扭矩(≠ 型錄的電測堵轉電流×Kt)
b = Kt²/R           ── 直線斜率絕對值(與電壓無關,只由 Kt、R 決定)
```

### 1.3 斜率受什麼設計參數影響?

斜率 `b = Kt²/R = R/(Kt·Ke) ≈ R/Kt²` 的物理意義是「馬達剛性(Stiffness)」:
- **斜率越小(直線越平緩)**:加上負載(轉矩增大)時轉速掉得少,馬達越「有力」
- **斜率越大(直線越陡)**:稍加負載轉速就大幅下降

影響斜率的三個設計參數:

| 參數 | 影響 | 設計含義 |
|---|---|---|
| 線圈內阻 R | R↑ → 斜率↑(更陡) | 線徑粗、匝數少 → R↓ → 特性更平緩 |
| 磁通量 Φ(磁鐵強度) | Φ↑ → Kt↑ → 斜率↓(更平緩) | NdFeB 稀土磁鐵提高 Kt |
| 線圈幾何 | 有效受力面積↑ → Kt↑ | 轉子直徑/長度↑(如 8520 比 716 更大 Kt) |

CL-8020 KV14.9 的 R=0.381Ω 極低,使其特性線非常平緩(剛性強),這正是它能在大槳負載下仍維持高轉速的關鍵。

### 1.4 參數從型錄反推

每一列型錄數據包含三個操作點,直接反推 Kt、R、I₀:

```
Kt ≈ V_nom / ω₀   (ω₀ 換算為 rad/s)
R  = V_nom / I_堵轉
I₀ = 型錄空載電流欄
```

驗證:CL-8020 KV14.9(V_nom=3.7V)
- Kt = 3.7/(55000×π/30) = 6.43×10⁻⁴ V·s/rad
- R = 3.7/9.70 = 0.381Ω  (與型錄端電阻 0.38Ω 吻合)
- 反算堵轉扭矩 a = 6.11 mN·m (型錄標 5.993 mN·m,差 2%,在量測誤差內)

## 2. 螺旋槳氣動負載模型

### 2.1 為什麼槳的 T-ω 是拋物線(與馬達相反)?

槳旋轉時受到空氣阻力轉矩(Aerodynamic Drag Torque):

- 槳葉某半徑 r 處的切向速度 = ω·r,所受阻力 ∝ 速度²
- 對整片槳葉從槳根積分到槳尖,總阻力矩 ∝ ω²

因此:**轉速愈快,反向阻力矩以平方倍率劇烈增大**——這就是槳的負載曲線往右上彎曲的原因,跟馬達的線性向右下降方向完全相反。

標準無因次係數定義(n=rev/s, D=直徑 m, ρ=空氣密度):
```
推力:F = CT·ρ·n²·D⁴
阻力矩:Q = CQ·ρ·n²·D⁵  (CQ = CP/2π)
```

以角速度 ω=2πn 表示:
```
Q_prop(ω) = k_Q·ω²,   k_Q = CP·ρ·D⁵/(4π²)
F_prop(ω) = k_T·ω²,   k_T = CT·ρ·D⁴/(4π²)
```

### 2.2 槳葉係數的工程數值

低雷諾數微型槳(45–65mm,Re<50,000)的代表性文獻係數(本文計算取 CT=0.10、CP=0.055):

| 槳徑 Φ | kf [N/(rad/s)²] | kt [N·m/(rad/s)²] | ctf=kt/kf [m] | Gemini文獻參考值 |
|---|---|---|---|---|
| 45mm | 1.25×10⁻⁸ | 4.91×10⁻¹¹ | 0.00394 | kf≈1.8×10⁻⁸, kt≈2.8×10⁻¹⁰ |
| 55mm | 2.78×10⁻⁸ | 1.34×10⁻¹⁰ | 0.00481 | kf≈2.4×10⁻⁸, kt≈4.3×10⁻¹⁰ |
| 65mm | 5.43×10⁻⁸ | 3.09×10⁻¹⁰ | 0.00569 | kf≈3.1×10⁻⁸, kt≈6.2×10⁻¹⁰ |

> ⚠️ 我們的 CT/CP 假設與 Gemini 文獻引用值有 30–50% 差距,原因是微型槳的 CT、CP 因葉片型號、螺距、涵道效應差異顯著。ctf(反扭矩-推力比)是飛控 Mixer Matrix 的關鍵參數,需實測確認。詳見第 11 節。

### 2.3 為何槳的負載曲線不受電壓影響?

這是最容易被誤解的關鍵點——槳的阻力轉矩 Q = k_Q·ω² 只由**轉速本身**決定,跟驅動它的電壓毫無關係。即使斷電,只要有外力讓這片槳轉到 30k rpm,它受到的空氣阻力矩和通電時完全相同。因此在圖上:**電壓改變時,三條槳葉拋物線位置完全固定不動。**

## 3. 平衡工作點——唯一穩定的交點

### 3.1 為什麼交點只有一個?

把馬達特性線(T-ω 直線,斜率為負)與槳負載曲線(Q-ω 拋物線,斜率為正且加速增大)畫在同一張圖上:
- 馬達線:左上到右下,轉速越低扭矩越大
- 槳負載線:從原點出發,轉速越高阻力矩越大(以平方倍率)

兩條「走向相反」的曲線在第一象限**有且僅有一個交點**——這就是系統會自動穩定的平衡工作點。

### 3.2 平衡點的物理穩定性論證

設系統當前轉速略低於交點:
- 馬達輸出扭矩 > 槳阻力矩 → 淨扭矩為正 → **系統加速**

設系統當前轉速略高於交點:
- 槳阻力矩 > 馬達輸出扭矩 → 淨扭矩為負 → **系統減速**

兩種情況都朝交點收斂,這個平衡點是**Lyapunov 穩定**的。系統一定會自動找到這個唯一的工作點。

### 3.3 平衡工作點方程

令馬達輸出扭矩 = 槳的需求扭矩,以 ω 為自變數:
```
a − b·ω = k_Q·ω²
→ k_Q·ω² + b·ω − a = 0
→ ω* = [−b + √(b² + 4·k_Q·a)] / (2·k_Q)
```

解出 ω* 後,所有系統量一次求得:
```
實際推力:F* = k_T·ω*²
實際電流:I* = (V − Kt·ω*)/R
電機效率:η* = (k_Q·ω*²·ω*) / (V·I*)
銅損熱耗:P_loss = I*²·R
推重比:T/W = 4·F* / (m·g)
```

> **這個 (ω*, F*, I*, η*) 才是設計基準——不是型錄的最大效率點。**

## 4. 效率分析與天花板

### 4.1 馬達自身的效率上限(封閉解)

設 u = ω/ω₀(轉速相對空載轉速的比例),γ = I₀·R/V(摩擦電流占堵轉電流的比例),代入效率公式整理:

```
η(u) = u − γ·u/(1−u)

令 dη/du = 0:
u* = 1 − √γ
η_max = (1 − √γ)²
```

這個 η_max 是馬達在這個電壓下**自身曲線**的絕對效率上限,跟裝什麼槳無關——型錄的「最大效率點」就是這個公式算出的值(在型錄測試電壓=額定電壓時精確吻合)。

**驗證**:CL-8020 KV14.9(γ = 0.187×0.381/3.7 = 0.0193):
```
u* = 1 − √0.0193 = 86.1%
η_max = 0.861² = 74.13%  ← 完全等於型錄標示的 74.13%(零誤差)
```

### 4.2 電壓下降時效率天花板本身也會降低

Gemini 文件(頁 8)只說最大效率點「平行左移」,但遺漏了一個重要事實:
當電壓 V 下降時,γ = I₀·R/V **變大**,η_max = (1−√γ)² 也**隨之下降**——不只是位置移動,連能達到的最高效率天花板都變低了。

以 CL-8520 KV13.7 為例(Kt=6.97×10⁻⁴,R=0.828Ω,I₀=102mA,均為物理常數):

| 電壓 V | γ=I₀R/V | u*=1−√γ | η_max | ω₀(rpm) | T_stall(mN·m) |
|---|---|---|---|---|---|
| 3.7V(滿電) | 0.0228 | 84.9% | **72.1%** | 50,700 | 3.044 |
| 3.3V | 0.0256 | 84.0% | **70.6%** | 45,219 | 2.707 |
| 3.0V | 0.0281 | 83.2% | **69.3%** | 41,108 | 2.455 |
| 2.5V(放電末) | 0.0338 | 81.6% | **66.6%** | 34,257 | 2.034 |

PWM 降低油門、或電池放電過程中,馬達線整體向左下**平行**移動(Kt/R/I₀不變,b不變,a隨V線性縮小),三條槳的拋物線固定不動,三個平衡交點各自沿著自己的拋物線滑向左下方——轉速、推力、電流同步下降。這正是油門控制的物理本質。

## 5. 正確設計流程(Top-Down 四步法)

### 步驟 1:確定推重比(TWR)

| 飛行用途 | 建議 TWR | 說明 |
|---|---|---|
| 室內教學/穩定型 | 1.5–1.8 | 餘量小,易控制,飛行時間長 |
| 標準航拍/室外 | ≈2.0 | 公認的「黃金比例」 |
| 穿越/3D特技 | ≥3.0 | 需要快速響應與抗風能力 |

以 55–60g 為例:
- 懸停推力(每軸) = m×g/4 = 60g/4 = **15g**
- 最大推力(每軸) = TWR×m×g/4 = 2.0×60g/4 = **30g**

### 步驟 2:槳徑尺寸與軸距幾何約束

X 型機架的槳間距約束:
```
軸距(對角線) > √2 · Φ_槳 + 安全間隙(建議 ≥ 10mm)
```

| 槳徑 | 最小對角軸距 | 常見機架 |
|---|---|---|
| 45mm | ≈74mm | 75mm Whoop |
| 55mm | ≈88mm | 85mm 圈圈機 |
| 65mm | ≈102mm | 100–110mm |

120mm 軸距機架理論上可容納最大約 80mm 槳葉(實際建議 ≤65mm)。

### 步驟 3:解平衡方程,求 ω*、F*

對每個候選「馬達×槳×電壓」組合,代入第 3.3 節的二次方程直接求解,不要靠估算或拍腦袋。

### 步驟 4:Stall 檢驗與熱安全

- **推力檢驗**:4·F* ≥ m·g·TWR_target
- **堵轉裕量**:I*/I_stall 應遠低於 1(建議 ≤ 30% 用於連續飛行)
- **銅損檢查**:P_loss = I*²·R 長時間超過 2–3W 對這個尺寸的空芯杯馬達風險很高

## 6. 額定電壓的正確解讀

型錄的「額定電壓 V_nom」只是廠商**選擇發表三個測試點數據時**所使用的單一測試電壓,不是唯一合法工作電壓——真正的限制是「工作電壓範圍」欄。

**跨電壓外推的可信度**:Kt、R 是繞線物理常數,在工作電壓範圍內幾乎不變;I₀ 理論上可能隨轉速有二階變化,外推幅度越大誤差越大。

| 外推程度 | 例子 | 可信度評估 |
|---|---|---|
| 零外推(測試電壓=驅動電壓) | 8020 KV14.9 @3.7V | ✅ 型錄數字直接可用 |
| 小幅外推(≤25%) | 8520 KV14.0 @3.7V(+23%) | ⚠️ 物理上合理,建議實測核對 |
| 大幅外推(>40%) | 8520 KV16.0 @3.7V(+54%) | ❌ 模型不確定性大,不建議信任 |

## 10. 模型限制(務必注意)

1. **CT/CP 假設不等於實際槳**:本文對 45–65mm 統一取 CT=0.10、CP=0.055,但真實小槳因螺距、葉片數、涵道效應差異可達 ±40%。直徑越小 Re 越低,氣動效率越差,「45mm 最好」的結論在氣動側可能被高估。
2. **電壓外推風險**:Kt、R 是物理常數,外推可靠;I₀ 有二階效應,幅度越大誤差越大(詳見第 6 節分級評估)。
3. **電壓下垂**:1S 從 4.2V 放到 3.5V,推力約下降 17%,電流下降 16%,效率天花板下降約 5 個百分點(見第 4.2 節)。長時間懸停應把最低工作電壓代入重算。
4. **涵道效應**:55mm 槳用於 Whoop 圈圈機時,保護圈形成類涵道效應,推力係數 CT 可提升 15–25%,但 CQ 也對應變化。
5. **槳葉個體差異**:同尺寸不同品牌、螺距(1.0–1.9 吋均有)的槳,kt 差距可接近一倍。
6. **連續工作熱安全**:空芯杯馬達的連續額定電流遠低於堵轉電流,I*/I_stall 建議 ≤ 30%(CL-8020 KV14.9 + 45mm 的比例僅 19%,最安全)。

## 11. 如何獲得真實槳葉係數(kT、kQ)

Gemini(頁 25-27)提到可用「葉素理論 BET」估算,但對低 Re 微型槳(Re<50,000)不準確,推薦以下工程做法:

### 方法 A:繫留測試臺逆向推導(最推薦)

1. 固定電壓,執行 5–8 個 PWM 階梯
2. 每個階梯穩定後記錄:RPM、電流 I、推力 F
3. 由電流算軸端扭矩:Q = Kt·(I − I₀)
4. 擬合: k_Q = Q/ω²,k_T = F/ω²
5. 驗算 ctf = k_Q/k_T(應等於 Gemini 表中的 ctf 數量級)

### 方法 B:CFD 數值模擬

將槳葉 3D STL 匯入 OpenFOAM/Ansys Fluent,旋轉流場計算可把黏滯力與槳尖渦流完全計入,精度最高但耗時。

## 附錄 B:Python 可重現計算程式

```python
import math

rho, CT, CP = 1.2, 0.10, 0.055
CQ = CP / (2*math.pi)
V_drive = 3.7
m, g = 0.055, 9.81
Weight = m*g

motors = [  # (series, KV, Vnom, Vmin, Vmax, I0_mA, w0rpm@Vnom, I_stall@Vnom)
    ("8520","11.9",3.7,1.5,4.0,92.0,44000,4.61),
    ("8520","13.7",3.7,1.5,4.0,102.0,50700,4.47),
    ("8020","14.9",3.7,1.5,4.0,187.5,55000,9.70),
]

def kTkQ(D):
    kT = CT*rho*(D**4)/(4*math.pi**2)
    kQ = CQ*rho*(D**5)/(4*math.pi**2)
    return kT, kQ

for series, kv, Vnom, Vmin, Vmax, I0_mA, w0rpm, Is in motors:
    if not (Vmin <= V_drive <= Vmax): continue
    I0, Kt = I0_mA/1000, Vnom/(w0rpm*math.pi/30)
    R = Vnom/Is
    a = Kt*(V_drive/R - I0); b = (Kt**2)/R
    gamma = I0*R/V_drive
    print(f"\n{series}-{kv}: η_ceiling={(1-math.sqrt(gamma))**2*100:.1f}%")
    for Dmm in [45, 50, 55, 60, 65]:
        D = Dmm/1000
        kT, kQ = kTkQ(D)
        w = (-b + math.sqrt(b*b + 4*kQ*a)) / (2*kQ)
        if w > V_drive/Kt: continue
        F = kT*w*w; I = (V_drive - Kt*w)/R
        eta = (kQ*w**3)/(V_drive*I)
        print(f"  Φ{Dmm}mm: {w*30/math.pi:.0f}rpm  {F/0.00980665:.1f}gf  "
              f"I={I:.2f}A  η={eta*100:.1f}%  T/W={4*F/Weight:.2f}")
```

## 附錄 C:如何看懂 Coreless Motor 特性曲線型錄

### C.1 這張圖在量什麼

廠商把馬達架在測功機上,固定電壓,用可調機械刹車從空載掃到堵轉,逐點記錄轉速與電流,再計算輸出功率與效率。

**關鍵認知**:橫軸的扭矩 T 是測功機外加的任意機械負載,不是任何特定螺旋槳的負載。槳的負載 Q=k_Q·ω² 是通過原點的拋物線——型錄圖上不會畫這條線,你必須自己疊上去,交點才是真正的運轉平衡點。

### C.2 雙 Y 軸的換算陷阱

右軸的 0–100% 同時給 Eff%、Watts、Amp 三條曲線共用,但「滿格代表的實際值」各自不同:
```
Eff 曲線:右軸刻度本身就是百分比,直接讀
Watts 曲線:實際 W = (右軸高度%) × Full Scale Watts
Amp 曲線: 實際 A = (右軸高度%) × Full Scale Amp
```
**只有 Eff 本身就是百分比**,Watts 和 Amp 都必須乘回 Full Scale 才是真實值——這是初學者最常犯的錯誤。

### C.3 四條曲線對應的方程式

| 曲線 | 形狀 | 對應方程 | 關鍵特徵 |
|---|---|---|---|
| 轉速(藍,直線) | 右下傾斜 | ω(T)=(a−T)/b | y截距=ω₀=V/Kt,x截距=T_stall=a |
| 電流(黑,直線) | 右上傾斜 | I(T)=T/Kt+I₀ | y截距=I₀,斜率=1/Kt |
| 功率(綠,拋物線) | 開口向下 | P(T)=T·ω=(aT−T²)/b | 峰值永遠在 T=a/2 |
| 效率(洋紅,鐘形) | 單峰 | η(u)=u−γu/(1−u) | 峰值在 T*<a/2(比功率峰值更靠近空載端) |

### C.4 從圖直接反推參數(五步驟)

1. 讀速度線在 T=0 的高度(左軸)→ ω₀,算 Kt≈V/ω₀
2. 讀電流線在 T=0 的高度(右軸×Full Scale A)→ I₀
3. 讀速度線跌到 0 的橫軸位置 → T_stall=a;讀電流線同位置 → I_stall,算 R=V/I_stall
4. **驗證**:綠色功率峰值橫軸位置應=T_stall/2
5. **驗證**:η_max 讀值應=(1−√(I₀R/V))²

### C.5 常見誤讀陷阱

- 忘記乘 Full Scale → 把 Watts/Amp 的百分比直接當數值
- 把堵轉點當連續工作點 → 銅損 I²R 嚴重過熱
- 效率峰值和功率峰值混為一談 → 兩者位置不同
- 把「額定電壓」誤認為唯一電壓 → 詳見第 6 節
- 不同電壓的圖直接比較截距數值 → 需先統一換算到同一電壓
